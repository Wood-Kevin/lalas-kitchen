# Deferred Complexity

One line per deferred idea, with a short reason it was deferred. Parked is fine. Forgotten is not.

- Row clearers, immovable blockers, and other special piece behaviors — out of scope for V1 per CLAUDE.md; `type` field left in place as insurance, no logic built yet.
- Recipe box meta layer UI — out of scope for V1 per CLAUDE.md; engine's end-of-level summary event should exist eventually but nothing listens to it yet.
- Cloud asset delivery / per-skin CDN loading — irrelevant until a second skin is real.
- Multi-target or score-threshold objectives — V1 is move limit plus one collection target only.
- App Store "distinct product" layout variation — matters only once skin number two ships.
- Auto-reshuffle when a level's board runs out of legal moves mid-play — `applyMove` currently just never consumes a move on an illegal swap, which could theoretically soft-lock a player on a stuck board; not built this phase since it wasn't requested and needs a design call (when to trigger, whether it costs anything) beyond gameState.ts's current scope.
- What actually decrements `GameState.lives` during a level attempt — Phase 4 built the pause/reason/resume mechanics (`pauseReason: 'lives'`, `grantBonusLife`) and `applyMove` checks `lives <= 0` as a trigger, but nothing spends a life yet. Asked the user to pick between "moves-exhaustion auto-spends a life" vs. "fully independent resource, spend trigger built later" and got no response in time; proceeded with the latter (smaller, more reversible) and flagged it clearly in `engine/DECISIONS.md` rather than guessing the game-design rule silently. Needs a decision before this is real gameplay, not just correct mechanics.
- Real sprite art for lalas-kitchen — `skins/lalas-kitchen/sprites/` is still empty. The real `<Image>` pipeline is built (`components/spriteAsset.ts` + `skins/lalas-kitchen/spriteRegistry.ts`, see `components/NOTES.md`); this line is now purely about the art itself not existing yet, not a missing pipeline. Dropping a file in needs exactly one `require()` line added to `spriteRegistry.ts` — no other file changes. Sprite format is `.webp` (`config.json`, decided to close the native-SVG rendering gap — see `components/NOTES.md`), so real art should be exported as WebP.
- Per-step cascade animation (each chained match popping as its own visual beat) — not possible against the current engine API, since `applyMove` only returns the final settled board, not intermediate cascade snapshots. `Board.tsx` currently animates a single before/after diff instead, which still looks smooth but collapses multi-cascade chains into one continuous motion. Would need `applyMove`/`resolveCascades` to expose intermediate board states to fix.
- Literal edge-to-edge fill on all four sides of the board — with square tiles and an 8-row/6-col grid on a typical tall phone screen, tile size is bound by screen *width* (6 columns is the tighter constraint), so there's unavoidable vertical slack even at max tile size. `Board.tsx` now centers the grid in the full available height (see `components/NOTES.md`) rather than dumping all the slack below it, but it isn't literally edge-to-edge top and bottom unless tiles go non-square or the grid's row/col ratio is changed — neither of which was asked for.
